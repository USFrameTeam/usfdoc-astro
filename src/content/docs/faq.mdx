## 如何反馈bug

<Steps>

1. **若无法使用USF**

   检查您的Minecraft版本与usf版本是否匹配，详见[兼容表](/edition)，然后检查是否正确安装USF

2. **检查Bug是否修复**

   如果版本兼容，您应该查看[更新日志](/change-log)。每个版本更新时都会修复大量bug。如果该bug已在新版本中被修复，您应该更新到最新版本。如果没有提及，您可以反馈该bug。

3. **其它情况，需要反馈bug**

   - 进入游戏打开如下选项

   ![image](/upload/202402/20241220001.jpg)

   - 重复出现bug的操作
   - 将出现的报错截图发到群里，最好[在Github创建issue](https://github.com/USFrameTeam/Unknown-Server-Framework/issues/new)

</Steps>

## 初始化问题

### 如何获取管理员权限？

<Aside type="danger">
首次使用必须初始化！
</Aside>

<Tabs>
  <TabItem label="单人存档">
    在聊天栏输入：
    ```
    /function get_owner
    ```
  </TabItem>
  <TabItem label="服务器">
    方法一：在聊天栏输入（需要已获得服务器OP）：
    ```
    /function get_owner
    ```
    
    方法二：在服务器后台控制台输入：
    ```
    /scriptevent usf:get_owner 玩家名
    ```
  </TabItem>
</Tabs>

<Aside type="caution">
单人存档和服务器的 OP 权限不互通，需要分别设置！
</Aside>

### 为什么重置后 OP 失效？

USF 的 OP 权限与 Xbox 账户绑定。如果重置了插件，需要重新获取权限：

<Steps>

1. 在游戏内输入 `/function reset`

2. 在 **30秒内** 输入 `/reload`

3. 重新输入 `/function get_owner`

</Steps>

## 命令问题

### 输入命令没反应？

<Steps>

1. 检查命令前缀是否正确（默认为 `+`）

2. 检查该命令是否在配置中启用：
   - `+land` - 需开启 `land.able`
   - `+die` - 需开启 `game.kill`
   - `+tpr` - 需开启 `tp.per`
   - `+back` - 需开启 `tp.back`

3. 检查是否有相应权限

</Steps>

### 如何修改命令前缀？

在管理界面 → 插件设置 → 命令配置 中修改可用命令列表。

### 命令冷却时间太长？

传送相关命令默认冷却 30 秒，可在管理界面 → 插件设置 → 传送设置 中修改 `tp.down` 值。

## 领地问题

### 无法设置领地？

<CardGrid>
  <Card title="检查命令开启" icon="setting">
    确认插件设置中已开启 `+land` 命令
  </Card>
  <Card title="检查计分板" icon="warning">
    领地费用计分板名称是否正确，是否重复
  </Card>
  <Card title="检查选点" icon="map">
    确认已选择两个对角点（空手潜行点击方块）
  </Card>
</CardGrid>

### 领地内可以挖东西？

<Aside type="caution">
确认领地设置中圈地为**三维模式**，而非二维。
</Aside>

### 如何删除领地？

<Steps>

1. 主菜单 → 领地

2. 选择要删除的领地

3. 点击「删除领地」

</Steps>

或管理员在管理界面 → 管理玩家领地 中删除。

### 领地权限不生效？

检查以下设置：
- 访客权限（`other_per`）是否正确配置
- 成员权限（`mem_per`）是否正确配置
- 是否将玩家添加为领地成员

## 传送问题

### 传送请求没收到？

<Steps>

1. 确认对方在线

2. 检查传送系统是否开启

3. 确认没有超过冷却时间

4. 对方可能在其他界面中，需要关闭界面才能看到请求

</Steps>

### Home 点设置失败？

<Steps>

1. 主菜单 → 传送系统 → 个人传送点

2. 添加传送点时勾选「设为 Home」

3. 确保只有一个 Home 点（多个会覆盖）

</Steps>

### 随机传送范围太大/太小？

在管理界面 → 插件设置 → 传送设置 中修改 `tp.random_range` 值（默认 10000）。

## 商店问题

### 商店打不开？

确认插件设置中已开启商店功能（`store.able`）。

### 购买物品没有扣钱？

检查：
- 货币记分板名称是否正确
- 玩家是否有该记分板的分数
- 商品配置中的货币ID是否匹配

### 如何添加商品？

<Steps>

1. 输入 `+op` 打开管理界面

2. 插件设置 → 全局商店设置

3. 管理全局商店 → 添加商品

4. 设置物品ID、价格、数量等

</Steps>

## 聊天问题

### 聊天格式显示异常？

检查聊天格式配置中的变量是否正确：
- `/sender` - 发送者名称
- `/tag` - 聊天头衔
- `/text` - 聊天内容

### 头衔不显示？

<Steps>

1. 确认已正确添加标签：`/tag 玩家名 add usf.tag:§6[大佬]§r`

2. 检查聊天格式中是否包含 `/tag` 变量

3. 使用 `usf.tag:Reset` 重置后重新设置

</Steps>

### 被禁言了怎么办？

禁言会在设定时间后自动解除。管理员可在管理界面 → 屏蔽/禁言玩家 中手动解除。

## 策略文件问题

### 策略文件是什么？

策略文件（`usf:config_file`）是 USF 的自定义 UI 系统，可以创建交互式菜单界面。

### 如何获取策略文件？

- 创造模式物品栏中搜索「策略文件」
- 使用命令 `/give @s usf:config_file`

### 策略文件界面打不开？

<Steps>

1. 确认手持策略文件

2. 右键空气打开界面

3. 如果没有反应，检查是否已配置界面内容

</Steps>

### 如何绑定到方块/NPC？

- **绑定方块**：手持策略文件，潜行+右键方块
- **绑定NPC**：手持策略文件，右键NPC

## 兼容性问题

### 假人类 addon/插件冲突

绝大多数假人 addon 需要通过发送聊天信息来操作假人，这与 USF 的聊天系统冲突。

**解决方案：** 在插件设置中关闭 USF 聊天格式，或不使用头衔功能。

### LeviAntiCheat 插件冲突

LeviAntiCheat 插件可能导致 USF 栈溢出。

**解决方案：** 不使用 LeviAntiCheat 插件，或二选一。

</Aside>

## 服务端加载问题

<Aside type="danger" title="注意">
此部分教程解决办法可能会有不确定或者需要一定操作能力/信息检索能力的部分。虽然这里面涵盖了部分USFrameTeam成员帮助他人时遇到的常见问题及理论解决办法，但USFrameTeam不对该部分教程作任何适用性承诺，如遇到不明白的部分请询问AI或向其他人寻求帮助。
</Aside>

<Tabs>
  <TabItem label="中文文件夹问题">
    **问题描述：** 部分情况下(例如Linux Wine环境)，当behavior_packs文件夹内USF文件夹名字带有中文，USF可能不加载

    **解决办法：** 删除中文
  </TabItem>
  <TabItem label="存档内没有USF">
    **问题描述：** 当按照文档内教程或者Minecraft机制原因

    **问题原因：** 较新版本添加模组在MC私有目录的 `behavior_packs` 文件夹而非存档 `behavior_packs` 文件夹。在游戏本地可以正常使用USF,但一但直接导出世界没有USF(包括Windows端基岩版导出世界/世界模板、全平台基岩版直接复制存档文件夹)

    **解决方法：**
    - 检查behavior_packs文件夹内是否有USF文件，如果没有需要添加
    - 如果没有 `behavior_packs` 文件夹，你需要在存档目录新建一个 `behavior_packs` 文件夹，然后再在里面新建个名字辨识度高的文件夹(比如USF)
    - 之后将USF文件复制到 `behavior_packs/{你新建的那个}` 文件夹内，此时USF就应添加成功了
  </TabItem>
  <TabItem label="服务商问题">
    **3.1 简幻欢服务商**

    - 问题描述：在简幻欢服务商提供的服务器中，直接在存档的worlds_behavior_packs.json里添加USF时，USF不会加载(暂未确定)
    - 解决办法：下载存档，从游戏内导入USF

    **3.2 Aternos等免费服务商**

    - 问题描述：像aternos这类免费服务商对gametest框架做了特殊的处理，导致USF无法在aternos等服务商使用
    - 解决办法：只能建议更换服务商
  </TabItem>
  <TabItem label="其它情况">
    - 检查 `behavior_packs.json`，查看USF UUID是否正确，如果没有 `behavior_packs.json`，需要创建一个并按照json格式填写USF的UUID(需要在存档开启测试版API的前提下USF通过此方法才能正常导入，没有则需开启)
    - 如果没有 `behavior_packs` 文件夹，你需要在存档目录新建一个 `behavior_packs` 文件夹，然后再在里面新建个名字辨识度高的文件夹(比如USF)
    - 之后将USF文件复制到 `behavior_packs/{你新建的那个}` 文件夹内，此时USF就应添加成功了
  </TabItem>
</Tabs>

## 服务器不是OP/存档内不是OP

**现象：** 我在存档里获取了最高op，存档拷贝到服务器后，却不显示op/在服务器里是最高OP,存档下载导入后却不是OP

**原因：** BE版游戏账户数据在本地是局域网账户，当使用xbox认证后进入服务器，数据库并没有你的信息，所以本质上服务器/存档已经有最高op了，但你却不是。

<Tabs>
  <TabItem label="服务器解决办法">
    <Steps>

    1. 获取 __服务器__ op：`op <你的Xbox ID>`
    2. __进入服务器__，在服务器内输入 `/function reset`
    3. 在 __30秒内__ 在 __服务器内__ 输入 `/reload`
    4. 然后重新输入 `/function get_owner`

    </Steps>
  </TabItem>
  <TabItem label="存档解决办法">
    <Steps>

    1. __进入存档__，输入 `/function reset`
    2. 在 __30秒内__ 输入 `/reload`
    3. 然后重新输入 `/function get_owner`

    </Steps>
  </TabItem>
</Tabs>

## 配置问题

### 配置修改后不生效？

大部分配置修改后会即时生效，无需重启。如果不生效：
- 检查配置格式是否正确
- 尝试输入 `+cd` 刷新菜单
- 极少数情况需要重启服务器

### 如何备份配置？

配置存储在世界存档的动态属性中，备份存档即备份配置。

### 配置文件在哪里？

USF V2 使用游戏内动态属性存储配置，不使用外部文件。所有配置通过游戏内管理界面修改。

## 日志系统问题

### 日志功能是什么？

USF 可以记录服务器各种事件（聊天、死亡、方块操作等），并发送到外部日志服务器。

### 如何启用日志？

<Steps>

1. 部署日志服务器（需要支持 HTTP 接收）

2. 在管理界面 → 插件设置 → 日志设置

3. 开启日志功能，填写服务器地址

4. 选择要记录的日志类型

</Steps>

### 日志服务器连接失败？

- 检查日志服务器地址是否正确
- 确认日志服务器正在运行
- 检查网络连接
- USF 会自动重连（默认间隔 30 秒）

## 其它模组/插件兼容问题

### 假人类addon/插件

绝大多数的假人addon/插件需要通过发送聊天信息来对假人进行操作，但这与USF有一定冲突，可能导致USF或者假人不可用。

<Aside type="caution" title="注意事项">
如果你想要在服务器使用假人，你将不能使用USF的头衔，如:[主世界]玩家名
</Aside>

**解决方案：** 在插件设置中找到USF聊天格式，关闭

### LeviAntiCheat插件

目前已知ll服务端LeviAntiCheat插件可能会导致USF栈溢出，但由于LeviAntiCheat插件是闭源的，我们并不能确定相关问题。
介于服务端被watchdog关闭前是LeviAntiCheat在操作，我们暂且估计是LeviAntiCheat的部分操作可能导致USF的栈溢出。

**解决方案：** 不使用LeviAntiCheat插件或者不使用USF，加强服务器管理也可以避免这种情况

## 常见Q&A

### 为什么例如防偷箱、火焰蔓延、活塞等领地功能和玩家市场一直没有加入USF

SAPI做不到系列，没有相应接口或者功能短缺。如果你想要这些功能，还想要完整的，支持最新版本的生存体验≈没有（至少原版BDS是不能实现）。

<Aside type="tip">
对于领地，最好家离出生点远点加上日志功能辅助判断是谁破坏的领地。
</Aside>

### 为什么USF旧版本没有新版本功能？

- 第一，有些USF功能sapi旧版本不支持
- 第二，电脑端微软商店自动更新，之前USF无法适配

### 为什么文档不完善

在写了在写了，USF文档的编写维护人员(包括EarthDLL)大部分都是学生，没太多精力去编写和维护，甚至根本不会，如果想要的功能教程提给@dyf189

<Aside type="note">
如果你可以接受由AI生成的USF文档，可以访问[USF项目AI生成文档](https://zread.ai/USFrameTeam/Unknown-Server-Framework)。
</Aside>

### Github为什么USF源代码更新很慢

Github只是开源的地方，USF本身并没有做混淆，如果想要修改源码请遵守GPL3.0协议

### USF会不会停更？

迫于成员学业压力，USF很可能会停更，如果你想贡献USF，请查看[USF社区贡献者招募计划](https://github.com/USFrameTeam/.github/discussions/1)

### 一些常见问题去哪找

看文档看文档(也就是这)，不会的问USF群友，但不保证态度和24h在线
