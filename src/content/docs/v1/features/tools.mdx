## 小木斧(WorldEdit)

V1内置小木斧工具，供OP玩家进行区域操作。

### 获取方式

手持 **木斧** 即可进入小木斧模式。

### 使用方法

<Steps>

1. 手持木斧挖掘第一个方块(绿色点)
2. 挖掘第二个方块(红色点)
3. 长按空白处或右键打开菜单
4. 选择操作类型

</Steps>

### 功能菜单

| 功能 | 说明 |
|------|------|
| 移动红色点 | 移动终点位置 |
| 移动绿色点 | 移动起点位置 |
| 填充 | 填充选区方块 |
| 退出 | 退出小木斧模式 |

### 填充操作

选择"填充"后，输入方块ID即可填充选区。

**示例**：
- `minecraft:stone` - 石头
- `minecraft:air` - 清空区域
- `minecraft:grass` - 草方块

### 选区显示

选区会以粒子效果显示边框：
- 白色边框 - 选区范围
- 绿色点 - 起点
- 红色点 - 终点

## NPC交互系统

V1支持通过物品设置NPC交互页面。

### 操作组(USF-operation)

通过纸张物品设置操作组，实现连锁反应。

#### 创建操作组

<Steps>

1. 获取纸张物品
2. 设置Lore第一行为 `USF-operation`
3. 设置Lore第二行为起点操作ID
4. 后续Lore为JSON格式的操作数据

</Steps>

#### 操作类型

| 类型 | 说明 |
|------|------|
| command | 运行命令 |
| has_tag | 检测标签 |
| hand_item | 检测手持物品 |

#### 操作配置

```json
{
  "id": "operate1",
  "type": "command",
  "data": {
    "command": "say Hello",
    "goal": 0
  },
  "next": "operate2",
  "true": "operate3",
  "false": "operate4"
}
```

| 字段 | 说明 |
|------|------|
| id | 操作ID |
| type | 操作类型 |
| data | 操作数据 |
| next | 下一个操作 |
| true | 成功后的操作 |
| false | 失败后的操作 |

#### 命令执行者

| 值 | 说明 |
|----|------|
| 0 | 执行操作的玩家 |
| 1 | 主世界 |

### 页面组(USF-page)

通过纸张物品设置NPC交互页面。

#### 创建页面组

<Steps>

1. 获取纸张物品
2. 设置Lore第一行为 `USF-page`
3. 设置Lore第二行为起点页面ID
4. 后续Lore为JSON格式的页面数据

</Steps>

#### 页面数据结构

```json
{
  "id": "page1",
  "title": "页面标题",
  "body": ["内容行1", "内容行2"],
  "buttons": [
    {
      "text": "按钮文本",
      "click_type": 0,
      "page_id": "",
      "op_id": "",
      "op": {}
    }
  ]
}
```

#### 按钮响应类型

| 值 | 说明 |
|----|------|
| 0 | 无操作 |
| 1 | 执行操作 |
| 2 | 跳转页面 |

## 计分板侧边栏

### 功能配置

```json
{
  "ScoreBoard": {
    "id": "计分板ID",
    "text": [
      ["行1内容", "行2内容"],
      ["轮换内容1", "轮换内容2"]
    ],
    "time": 60
  }
}
```

### 配置说明

| 字段 | 说明 |
|------|------|
| id | 计分板ID |
| text | 显示内容(支持多组轮换) |
| time | 轮换间隔(秒) |

### 内容变量

支持以下变量替换：

| 变量 | 说明 |
|------|------|
| /name | 玩家名称 |
| /pos | 玩家位置 |
| /list | 在线玩家列表 |
| /alltime | 世界运行总时间 |
| /board.ID.score | 计分板分数 |
| /board.ID.name | 计分板显示名称 |
| /dimension | 玩家所在维度 |

## 公告系统

### 功能配置

公告内容存储在 `board_text` 数据中。

### 显示方式

- 主菜单中显示公告
- 支持多行内容
- 支持展开/收起

### 配置项

| 配置 | 说明 | 默认值 |
|------|------|--------|
| board.length | 不展开时显示行数 | 5 |
