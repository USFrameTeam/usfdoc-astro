## 权限管理

### 最高OP(Owner)

最高OP拥有所有权限，通过以下方式设置：

<Steps>

1. 运行命令 `/scriptevent usf:manager`
2. 首次运行的玩家自动成为最高OP
3. 后续运行会打开管理界面

</Steps>

### 重置最高OP

运行命令：
```
/scriptevent usf:reset
```

需在20秒内重启服务器或执行 `/reload` 命令。

### 普通OP管理

最高OP可通过OP界面管理普通OP：

- 开启/关闭OP权限
- 分配功能权限
- 查看OP列表

### OP功能权限

| 权限ID | 说明 |
|--------|------|
| Enable | 开启/关闭OP |
| Player | 玩家管理功能 |
| BanList | 封禁列表管理 |
| Score | 计分板修改 |
| BagCheck | 背包检查 |
| Setting | 修改插件设置 |

## 封禁系统

### 封禁列表

管理界面可管理封禁列表：
- 添加封禁玩家
- 移除封禁玩家
- 查看封禁列表

### 自动踢出

开启 `Kick` 功能后，封禁列表中的玩家会被自动踢出。

### 解封命令

```
/scriptevent usf:unban 玩家名
```

## 游戏规则锁定

开启 `Rule` 功能后，可锁定游戏规则。

### 可锁定规则

| 规则ID | 说明 |
|--------|------|
| mobGriefing | 生物破坏 |
| keepInventory | 死亡不掉落 |
| tntExplodes | TNT爆炸 |
| showCoordinates | 显示坐标 |
| pvp | PVP |
| doMobSpawning | 生物生成 |
| doImmediateRespawn | 立即重生 |
| commandBlocksEnabled | 使用命令方块 |

### 配置方式

```json
{
  "Rule": ["mobGriefing", "keepInventory"],
  "RuleOp": [false, true]
}
```

- `Rule` - 规则ID列表
- `RuleOp` - 对应的值(true/false)

## 模式锁定

### 锁定生存模式

开启 `LockMode` 功能后：
- 非OP玩家强制为生存模式
- 每20tick检测一次
- 创造模式玩家自动切换为生存

## 清洁工

开启 `Cleaner` 功能后，系统定时清理掉落物。

### 清理周期

- 每5分钟(6000tick)清理一次
- 清理前1分钟提示
- 清理前30秒倒计时

### 提示信息

| 时间 | 提示 |
|------|------|
| 清理前1分钟 | §e[清洁工]一分钟后清理所有掉落物 |
| 清理前30秒 | §e[清洁工]Xs后清理所有掉落物 |
| 清理完成 | §e[清洁工]已清理所有掉落物 |

### 清理规则

- 清理8格范围内无玩家的掉落物
- 有玩家附近的掉落物不会被清理

## 伤害提示

开启 `HurtTip` 功能后，攻击实体时显示血量信息。

### 显示内容

- 血量条(颜色渐变)
- 当前血量/最大血量
- 本次伤害值

### 显示格式

```
||||||||||||||||||||||||||||||
当前血量 / 最大血量 (♥-伤害值)
```

## 范围挖掘检测

开启 `RangeDig` 功能后，检测异常挖掘行为。

### 检测规则

- 2tick内挖掘4个方块触发
- 自动踢出玩家
- OP玩家豁免
- 创造模式豁免

## 密码箱

开启 `Chest` 功能后，可保护箱子不被他人打开。

### 保护规则

- 非OP玩家无法打开他人箱子
- 非OP玩家无法破坏他人箱子
- 使用NBT检测防止非法箱子

## 区块备份

开启 `Backup` 功能后，系统自动备份玩家所在区块。

### 备份周期

- 每分钟(1200tick)备份一次
- 备份玩家所在区块
- 使用结构文件存储

### 备份命名

```
backup_维度ID_(区块X_区块Z)_备份ID
```
