## 功能概述

V1队伍系统允许玩家创建和管理队伍，支持队伍邀请、成员管理等功能。

## 队伍权限

### 队伍等级

| 等级 | 说明 |
|------|------|
| 0 | 非队内成员 |
| 1 | 普通队员 |
| 2 | 队伍管理员 |
| 3 | 队伍创建者 |

### 权限说明

- **队伍创建者** - 拥有所有权限，可解散队伍
- **队伍管理员** - 可邀请/踢出成员
- **普通队员** - 基本队伍功能

## 队伍操作

### 创建队伍

通过主菜单 → 队伍系统 → 创建队伍

### 邀请成员

<Steps>

1. 打开队伍界面
2. 选择"邀请成员"
3. 选择要邀请的玩家
4. 对方收到邀请提示

</Steps>

### 接受邀请

玩家收到邀请后，前往队伍系统查看并接受。

### 邀请提示

```
§e玩家X邀请您加入队伍Y，请前往队伍系统查看！
```

## 队伍数据

### 数据存储

队伍数据存储在数据实体(Data Entity)的物品栏中。

### 数据格式

```json
{
  "队伍ID": {
    "op": ["创建者", "管理员1", "管理员2"],
    "mem": ["成员1", "成员2"]
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| op | 数组 | 管理员列表(第一个为创建者) |
| mem | 数组 | 普通成员列表 |

## 队伍与领地

队伍系统与领地系统联动：
- 领地可设置开放给特定队伍
- 队伍成员自动获得领地成员权限

## 队伍配置

### 队伍需求

可配置队伍相关的任务需求：

```json
{
  "team_require": {}
}
```

### 队伍数量限制

默认最多10个队伍，可通过配置调整。

## 队伍数据管理

### 保存机制

- 每分钟自动保存
- 服务器关闭时保存
- 数据存储在实体物品Lore中

### 加载机制

- 服务器启动时加载
- 数据实体创建后加载
- 支持断线重连恢复
