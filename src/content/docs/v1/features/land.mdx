## 功能概述

V1领地系统允许玩家创建和管理自己的领地，保护领地内的建筑和资源。

## 启用方式

在管理界面开启 `Land` 功能。

<Aside type="caution">
此功能标记为实验性，使用前会显示警告提示。
</Aside>

## 领地创建

### 使用工具

OP玩家使用 **木斧** 进行领地标记。

<Steps>

1. 手持木斧
2. 挖掘第一个方块标记起点(绿色点)
3. 挖掘第二个方块标记终点(红色点)
4. 长按空白处或右键打开菜单
5. 选择填充或其他操作

</Steps>

### 选区显示

选区会以粒子效果显示：
- **绿色点** - 起点位置
- **红色点** - 终点位置
- **白色边框** - 选区范围

## 领地权限

### 权限等级

| 等级 | 说明 |
|------|------|
| 0 | 无权限(被限制) |
| 1 | 成员权限 |
| 3 | 领地主人 |

### 权限判断逻辑

1. 如果玩家是领地主人 → 等级3
2. 如果玩家在开放玩家列表中 → 等级1
3. 如果玩家在开放队伍中 → 等级1
4. 否则 → 等级0

## 领地保护

### 保护机制

当无权限玩家进入领地时：
- 玩家会被限制移动
- 视角会被锁定
- 使用骡子实体实现位置锁定

### 领地实体

领地使用实体进行区域检测：
- **潜影弹(Shulker Bullet)** - 领地标记实体
- **骡子(Mule)** - 玩家限制实体

## 领地配置

### 领地数据结构

```json
{
  "di": "维度ID",
  "id": "领地ID",
  "name": "领地名称",
  "x": 0,
  "y": 0,
  "z": 0,
  "dx": 0,
  "dy": 0,
  "dz": 0,
  "owner": "主人名称",
  "member": ["成员列表"]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| di | 字符串 | 维度ID |
| id | 字符串 | 领地唯一ID |
| name | 字符串 | 领地显示名称 |
| x/y/z | 数字 | 领地起始坐标 |
| dx/dy/dz | 数字 | 领地结束坐标 |
| owner | 字符串 | 领地主人名称 |
| member | 数组 | 成员列表 |

## 队伍与领地

领地支持开放给特定队伍：

- 领地可设置开放队伍列表
- 队伍成员自动获得成员权限
- 支持多队伍同时授权

## 限制说明

- 领地范围不能超出已加载区块
- 超出范围会提示"领地范围超出加载区块，无法显示"
- 领地系统仍在开发中，功能可能不完整
