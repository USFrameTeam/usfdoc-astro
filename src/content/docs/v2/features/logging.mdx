日志系统可以记录服务器里发生的所有事情，方便管理员查案！

## 服主指南

### 功能配置

管理界面 → 插件设置 → 日志设置

| 配置项 | 说明 | 建议 |
|--------|------|------|
| 启用日志 | 功能开关 | 推荐开启 |
| 日志服务器地址 | 日志数据接收端 | 必须配置 |
| 重连间隔 | 连接断开后的重试周期 | 默认30秒即可 |

### 记录类型

管理界面 → 插件设置 → 日志设置 → 记录类型

| 记录类型 | 说明 | 建议 |
|----------|------|------|
| 聊天记录 | 玩家发言内容 | 推荐开启 |
| 进出记录 | 玩家上线/下线 | 推荐开启 |
| 破坏方块 | 方块挖掘操作 | 推荐开启 |
| 放置方块 | 方块放置操作 | 推荐开启 |
| 与方块交互 | 容器/工作台使用 | 按需开启 |
| 死亡记录 | 玩家死亡事件 | 推荐开启 |
| 击杀记录 | 实体击杀事件 | 按需开启 |
| 容器记录 | 物品存取操作 | 推荐开启 |
| 传送记录 | 传送行为 | 按需开启 |
| 维度切换 | 跨维度移动 | 按需开启 |
| 告示牌记录 | 告示牌内容变更 | 推荐开启 |
| 玩家位置 | 定时位置快照 | 按需开启 |

<Aside type="tip">
建议至少开启聊天、进出、破坏、放置、容器记录，基本覆盖常见管理需求。
</Aside>

## 日志服务器

日志需要一个外部服务器来接收和存储。

### 日志格式

日志以 HTTP 请求发送，包含：
- 事件类型
- 事件详情
- 玩家信息
- 时间戳
- 位置信息

### 连接失败怎么办？

- 检查日志服务器地址是否正确
- 确认日志服务器正在运行
- 检查网络连接
- USF 会自动重连（默认间隔 30 秒）

## 玩家指南

### 日志能记录什么？

| 记录类型 | 说明 |
|----------|------|
| 聊天记录 | 谁说了什么 |
| 进出记录 | 谁进来了、谁出去了 |
| 方块记录 | 谁挖了什么、放了什么 |
| 死亡记录 | 谁死了、被谁杀死的 |
| 容器记录 | 谁开了箱子、拿了什么东西 |
| 传送记录 | 谁传送到哪里 |
| 告示牌记录 | 谁改了告示牌内容 |

### 为什么需要日志？

减少纠纷,事后查询比较方便.

- **查熊孩子**：谁把你的房子拆了？一查便知！再也不用体验"苦力怕把你家炸了，你却不知道是谁干的"了！
- **找回物品**：谁拿了你箱子里的东西？有证据！再也不用体验"挖矿三小时，钻石零颗"了！
- **解决纠纷**：有证据就好说话，不用吵来吵去！

### 我能看到日志吗？

普通玩家看不到日志，只有管理员（OP）才能查看。

如果你怀疑有人搞破坏，可以找管理员帮你查！
