日志系统可以记录服务器里发生的所有事情，方便管理员查案！

## 服主看这里

### 开启日志功能

管理界面 → 插件设置 → 日志设置

| 配置 | 说明 | 建议 |
|------|------|------|
| 启用日志 | 开关 | 建议开 |
| 日志服务器地址 | 发送到哪里 | 必须设置 |
| 重连间隔 | 断线重连时间 | 默认30秒就好 |

### 选择记录类型

管理界面 → 插件设置 → 日志设置 → 记录类型

| 记录类型 | 说明 | 建议 |
|----------|------|------|
| 聊天记录 | 谁说了什么 | 建议开 |
| 进出记录 | 谁进来了、谁出去了 | 建议开 |
| 破坏方块 | 谁挖了什么 | 建议开 |
| 放置方块 | 谁放了什么 | 建议开 |
| 与方块交互 | 谁开了箱子 | 看需求 |
| 死亡记录 | 谁死了 | 建议开 |
| 击杀记录 | 谁杀了谁 | 看需求 |
| 容器记录 | 谁拿了什么东西 | 建议开 |
| 传送记录 | 谁传送到哪里 | 看需求 |
| 维度切换 | 谁去了下界/末地 | 看需求 |
| 告示牌记录 | 谁改了告示牌 | 建议开 |
| 玩家位置 | 每60秒记录一次 | 看需求 |

<Aside type="tip">
建议至少开启：聊天、进出、破坏、放置、容器记录，这样基本能查清楚大部分事情。
</Aside>

## 日志服务器

日志需要一个外部服务器来接收和存储。

### 日志格式

日志以 HTTP 请求发送，包含：
- 事件类型
- 事件详情
- 玩家信息
- 时间戳
- 位置信息

### 连接失败怎么办？

- 检查日志服务器地址是否正确
- 确认日志服务器正在运行
- 检查网络连接
- USF 会自动重连（默认间隔 30 秒）

## 玩家看这里

### 日志能记录什么？

| 记录类型 | 说明 |
|----------|------|
| 聊天记录 | 谁说了什么 |
| 进出记录 | 谁进来了、谁出去了 |
| 方块记录 | 谁挖了什么、放了什么 |
| 死亡记录 | 谁死了、被谁杀死的 |
| 容器记录 | 谁开了箱子、拿了什么东西 |
| 传送记录 | 谁传送到哪里 |
| 告示牌记录 | 谁改了告示牌内容 |

### 为什么需要日志？

你辛辛苦苦盖了个房子，结果出门挖矿回来发现被熊孩子拆了？或者你箱子里的钻石不见了？日志系统就是为了解决这个问题！

- **查熊孩子**：谁把你的房子拆了？一查便知！再也不用体验"苦力怕把你家炸了，你却不知道是谁干的"了！
- **找回物品**：谁拿了你箱子里的东西？有证据！再也不用体验"挖矿三小时，钻石零颗"了！
- **解决纠纷**：有证据就好说话，不用吵来吵去！

### 我能看到日志吗？

普通玩家看不到日志，只有管理员（OP）才能查看。

如果你怀疑有人搞破坏，可以找管理员帮你查！
